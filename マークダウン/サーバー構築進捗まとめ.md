# 漫画ライブラリ サーバー構築 進捗まとめ

## 構成概要

```
ポートフォリオ戦略:
  ├── Render（無料）         → Node.js/Expressサーバー デプロイ先
  ├── GitHub                → コード公開・バージョン管理
  └── Mac ローカル          → 本物の漫画アプリをデモ用に持参
```

---

## 実装済みセキュリティ対策

| 対策 | 実装方法 |
|------|---------|
| JWT認証 | `jsonwebtoken` / httpOnly Cookie |
| パスワードハッシュ | `bcryptjs`（saltRounds=12） |
| レートリミット | `express-rate-limit`（一般:100req/15min、ログイン:10req/15min） |
| HTTPヘッダー強化 | `helmet.js`（CSP / X-Frame-Options / HSTS等） |
| ディレクトリトラバーサル対策 | `path.resolve()`でベースDIR外アクセスを遮断 |
| セッションハイジャック対策 | httpOnly + Secure + SameSite=Strict |
| SQLインジェクション対策 | プリペアドステートメント（`better-sqlite3`） |
| HTTPS | デプロイ先（Render）が自動対応 |

## 環境構築の手順（実施済み）

### 1. Node.js バージョン確認
```
node -v  → v24.13.0
npm -v   → 11.6.2
```

### 2. フォルダ構成を整備
- `server-manga-viewer/` フォルダを作成
- `web/` フォルダを作成して既存フロントエンドファイルを配置
- `web/lib/` に `pdf.min.js` / `pdf.worker.min.js` を配置

### 3. Visual Studio Build Tools インストール
- `better-sqlite3`（C++製ネイティブモジュール）のビルドに必要
- 「C++ によるデスクトップ開発」ワークロードを選択してインストール
- インストール先: `C:\Program Files (x86)\Microsoft Visual Studio\2022\BuildTools`

### 4. npm install
- `better-sqlite3 v9` → Node.js v24未対応でエラー
- `better-sqlite3@latest`（v11）にアップデートして解決

```bash
npm install
npm install better-sqlite3@latest
```

### 5. JWT_SECRET 生成
```bash
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
```
生成されたキーを`.env`に設定済み

---

## .env の内容

```
JWT_SECRET=（生成した64文字のランダム文字列）
PORT=8000
NODE_ENV=development
MANGA_DIR=./manga_data
```
